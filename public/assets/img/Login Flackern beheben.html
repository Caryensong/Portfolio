Das Problem ist nicht direkt ein „Fehler“, sondern eher ein UI-Flackern, das auftritt, weil der Login-Zustand erst nach dem Laden überprüft und dann umgeleitet wird.

💡 Was passiert genau?

  1. Beim Page Refresh wird deine Angular App neu gestartet.

  2. Der Router lädt zuerst die Startkomponente (MainContentComponent, da dein Pfad / ist).

  3. Wenn du z. B. in der AppComponent oder MainContentComponent nicht sofort prüfst, ob der User eingeloggt ist, wird erstmal standardmäßig die Login-Komponente angezeigt.

  4. Danach prüfst du (vermutlich irgendwo in deinem Code) den Login-Zustand via Firebase, und wenn gültig, leitest du den User zur /summary Seite weiter.

  5. Dieses kleine Zeitfenster erzeugt den sichtbaren „Jump“ zur Login-Seite.

✅ Lösungsmöglichkeiten

1. Authentifizierungsstatus direkt beim Start prüfen

In deiner AppComponent oder MainContentComponent solltest du sobald wie möglich prüfen, ob der User eingeloggt ist, z. B.:

ts
import { Auth, onAuthStateChanged } from '@angular/fire/auth';
import { Router } from '@angular/router';

constructor(private auth: Auth, private router: Router) {}

ngOnInit() {
  onAuthStateChanged(this.auth, (user) => {
    if (user) {
      // Benutzer ist eingeloggt
      this.router.navigate(['/summary']);
    } else {
      // Benutzer ist nicht eingeloggt
      this.router.navigate(['/log-in']);
    }
  });
}

💡 Alternativ kannst du den aktuellen User auch direkt über this.auth.currentUser oder authState überprüfen, aber onAuthStateChanged() ist reaktiver.

2. Route Guards verwenden (empfohlen!)

Ein AuthGuard verhindert das Laden bestimmter Seiten, wenn der Benutzer nicht eingeloggt ist – so vermeidest du, dass die Login-Seite unnötig geladen wird.

Beispiel:

ts
// auth.guard.ts
import { Injectable } from '@angular/core';
import { CanActivate, Router } from '@angular/router';
import { Auth } from '@angular/fire/auth';
import { map } from 'rxjs/operators';
import { inject } from '@angular/core';
import { authState } from '@angular/fire/auth';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  auth = inject(Auth);
  router = inject(Router);

  canActivate() {
    return authState(this.auth).pipe(
      map(user => {
        if (user) {
          return true;
        } else {
          this.router.navigate(['/log-in']);
          return false;
        }
      })
    );
  }
}

Dann in deinen Routes:

ts
{ path: 'summary', component: SummaryComponent, canActivate: [AuthGuard] },

3. Loading-Indicator beim Initialisieren

Falls du noch immer ein kurzes Flackern hast, kannst du zusätzlich ein isLoading-Flag setzen und während des Auth-Checks einen Loader anzeigen.

html
<div *ngIf="isLoading">Loading...</div>
<app-log-in *ngIf="!isLoading && !user"></app-log-in>
<router-outlet *ngIf="!isLoading && user"></router-outlet>

Fazit

Das kurze Anzeigen des Login-Screens kommt daher, dass du erst nach dem Laden der Seite den Authentifizierungsstatus überprüfst. Am besten löst du das mit:

  • einem AuthGuard, der direkt beim Routing entscheidet,

  • oder mit einem frühen Auth-Check beim App-Start (z. B. in AppComponent),

  • und optional einem Loader, bis der Zustand geklärt ist.

import { Component, Inject, inject, PLATFORM_ID, OnInit } from '@angular/core';
import { Router, RouterModule, RouterOutlet } from '@angular/router';
import { FirebaseService } from './shared/service/firebase.service';
import { SharedComponent } from "./shared/shared.component";
import { LogInComponent } from './log-in/log-in.component';
import { SignUpComponent } from "./sign-up/sign-up.component";
import { Auth, onAuthStateChanged, signOut } from '@angular/fire/auth';
import { isPlatformBrowser } from '@angular/common';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [RouterOutlet, SharedComponent, RouterModule, LogInComponent, SignUpComponent],
  templateUrl: './app.component.html',
  styleUrl: './app.component.scss'
})
export class AppComponent implements OnInit {
  auth = inject(Auth);
  title = 'join';
  firebase = inject(FirebaseService);
  isLoggedIn = false;
  newUser?: boolean;
  usersName?: string;
  currentRoute: string = '';
  isLoading = true;

  constructor(private router: Router, @Inject(PLATFORM_ID) private platformId: Object) {
    this.router.events.subscribe(() => {
      this.currentRoute = this.router.url;
    });
  }

  ngOnInit() {
    if (isPlatformBrowser(this.platformId)) {
      onAuthStateChanged(this.auth, (user) => {
        this.isLoggedIn = !!user;
        this.isLoading = false;

        if (this.isLoggedIn && this.router.url === '/') {
          this.router.navigate(['/summary']);
        }
      });
    }
  }

  isPrivacyOrImprintPage(): boolean {
    return this.currentRoute === '/imprint' || this.currentRoute === '/privacy-policy';
  }

  onLoginSuccess() {
    this.isLoggedIn = true;
    if (isPlatformBrowser(this.platformId)) {
      localStorage.setItem('isLoggedIn', 'true');
    }
    this.router.navigate(['/summary']);
  }

  goToSignUp() {
    this.newUser = true;
  }

  logout() {
    signOut(this.auth).then(() => {
      this.isLoggedIn = false;
      localStorage.removeItem('isLoggedIn');
      this.router.navigate(['/']);
    }).catch(console.error);
  }

  resetNewUser() {
    this.newUser = false;
    this.isLoggedIn = false;
  }
}
